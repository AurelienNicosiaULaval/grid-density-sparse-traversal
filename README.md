# Exact sparse traversal for local finite-stencil grid-count density estimators

This repository contains the manuscript, R code, C++ backend, benchmark outputs,
and reproducibility materials for the paper:

Exact sparse traversal for local finite-stencil grid-count density estimators

The project studies exact sparse-prefix traversal for existing local
finite-stencil grid-count density estimators. It does not define a new density
estimator. The computation leaves the estimator, weights, bin widths, and
boundary conventions unchanged, and prunes only stencil branches whose grid
counts are exactly zero.

All numerical examples use synthetic data generated by the scripts in this
repository. No external data are required.

## Requirements

The numerical run reported in the manuscript used R 4.5.0. The main scripts use:

- `Rcpp`
- `RcppParallel`
- `ggplot2`
- `sessioninfo`

The lightweight test script also uses `testthat`.

The compiled sparse-prefix backend requires a C++17 compiler. The manuscript
requires a LaTeX installation with `latexmk`, `pdflatex`, and BibTeX. The
Springer Nature LaTeX class and bibliography style files are included in
`manuscript/`.

## Compile the manuscript

From the repository root:

```bash
cd manuscript
latexmk -pdf main.tex
```

If `latexmk` is unavailable:

```bash
cd manuscript
pdflatex main.tex
bibtex main
pdflatex main.tex
pdflatex main.tex
```

The compiled PDF is written to:

```text
manuscript/main.pdf
```

## Reproduce the numerical study

Run the full reproducibility pipeline with:

```bash
Rscript code/scripts/run_all_benchmarks.R
```

This runs correctness checks, estimator-family benchmarks, scaling benchmarks,
occupancy-controlled benchmarks, mode-selector benchmarks, figure generation,
table traceability checks, and session-information export.

The full pipeline regenerates timing CSV files. For a smoke run that reuses the
existing timing CSV files, use:

```bash
RUN_FULL=FALSE Rscript code/scripts/run_all_benchmarks.R
```

This smoke mode runs correctness checks, regenerates figures from existing CSV
files, checks table traceability, and writes session information.
